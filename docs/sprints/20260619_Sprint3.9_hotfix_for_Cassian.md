## 一、Hotfix 細項

### H0 — 出生城市 / 配偶出生地欄位底色
- **症狀**：輸入框背景色與文字色衝突，文字完全看不見
- **位置推測**：`public/index.html` 或登入後表單模組的 CSS
- **修法**：與其他欄位（出生日期、性別等）統一 input class；確認 `background-color` 與 `color` 套用同一組 design token

### H1 — 登入後分類欄寬度
- **症狀**：
  - 登入 Google 帳號後，分類區塊橫向寬度過寬，不符合上下區塊比例
  - 送出資料後，最上方的分類欄位也是相同狀況
- **位置推測**：登入後渲染分類列的 JS 模組（`public/js/categories.js` 或 chart-app.js 內片段）
- **修法**：分類欄套用與上下區塊一致的 `max-width` 或 grid column 對齊

### H2 — 格局、雙干疊加欄位底色
- **症狀**：底色與整體 UI 不協調，需改為「適合整體設計的淺咖啡色 + 對比文字色」
- **修法**：套用 `public/css/aesthetic.css` 中既有淺咖啡 token（若無則新增 `--brown-light`）；文字色從底色中突出（建議深咖啡或白字）

### H3 — 格局文字 + 配色（與 Sprint 4 P3 連動）
- **症狀**：
  - 文字用簡稱（要全名：機月同梁格、空劫入命格 …）
  - 區塊顏色用紅 → 改用整體 UI 一致色但加重 highlight
  - 多個格局時採交互色：
    - 第 1 格：beige 底 + 深灰字
    - 第 2 格：深咖啡底 + 白字
    - 第 3 格：同第 1 格配色
    - 依此類推
- **整合策略**：
  - Sprint 4 Spec v4.0 §四 已列「格局徽章」為 P3
  - 建議 **不單獨 hotfix**，併入 Sprint 4 P3 一起做（節省一次完整 design pass）
  - 若 Cassian 評估 P3 開工前還有時間，可先做純樣式部分

### H4 — 儲存命例 & 分類呈現（兩件事）
- **症狀**：
  - 登入並輸入資料後，**沒有「選擇分類後儲存」按鍵**
  - 登入後點擊分類時，**看不到已儲存到該分類的命例**
- **位置推測**：`public/js/library-store.js` 或 owner-ext 命例庫模組（Drive appData 寫入/讀取邏輯）
- **修法**：
  1. 在輸入完命主資料 → 起盤後的結果頁，新增「儲存到分類」按鈕（下拉選分類 → 確認）
  2. 寫入 Drive appData 對應子資料夾（樹狀結構參考 H6）
  3. 分類列 onClick → fetch appData 對應目錄 → 渲染命例 list（含命主名、生日、性別、起盤時間戳）
- **依存**：H6 的樹狀結構需先確認

### H5 — 合盤入口
- **症狀**：登入並輸入 Blue + Sean 雙人資料後，**沒看到合盤的 UI 或跳轉入口**
- **位置推測**：chart.html 結果頁未加合盤導向按鈕；舊版 `public/synastry.html` 雖存在但未串接
- **修法（hotfix 階段）**：
  - 結果頁加「查看合盤」按鈕，**僅在雙人都已輸入時顯示**
  - onClick → 導 `synastry.html?selfId=<id>&partnerId=<id>`（用 H4 寫入的命例 id）
  - 暫不重做 synastry.html 視覺（這是 Sprint 4 P0 範圍），僅打通入口
- **完整合盤體驗**：等 Sprint 4 P0 交付（雙盤並排 + 中央交叉飛化矩陣）

### H6 — Google Drive appData 樹狀資料夾
- **症狀**：appData 隱藏資料夾的樹狀結構是否已建立未確認
- **行動**：Cassian 確認當前 appData 狀態，若未建立：
  - 提供 terminal-runnable 指令（gdrive CLI 或 owner-ext 既有初始化 script）
  - 樹狀結構建議（待 Blue 確認分類命名）：
    ```
    appData/
    ├── self/              # 命主本人命例
    ├── partner/           # 配偶/伴侶命例
    ├── family/            # 家人命例
    ├── client/            # 客戶命例
    └── case_study/        # 教學案例（如 SARAJ）
    ```
- **交付**：可直接複製到 terminal 執行的初始化指令
